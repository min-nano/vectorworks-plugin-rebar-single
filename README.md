# vectorworks-plugin-rebar-single

3D パスに沿って **1 本の鉄筋** を配置する VectorWorks **プラグインオブジェクト（PIO）** スクリプトです。3D パス（鉄筋の芯線）と呼び径（`D13` 等）を保持し、次の 3 系統を出力します。

- **本体（3D）**: 鉄筋径（最外径）の円形断面をパスに沿って押し出した丸鋼のソリッド。
- **平面ビュー**: 上から見たパスの投影図（2D 線）。
- **断面記号（3D ソリッド）**: 呼び径に応じた**表示記号**（●／× 等、配筋標準図 KSE 2008「鉄筋の表示記号」）の**断面形状**をパスに沿って押し出したソリッド。断面ビューポートがこの 3D ソリッドを**ネイティブに切断**するため、折れ・フック・斜め配置でも、どの位置で切っても正しい位置に記号が出ます。

記号ソリッドは本体とは**別のクラス**（`SymbolClass` パラメータで指定）に割り当てます。ビューポートごとにクラス表示を切り替えて、**3D ビューでは本体（丸鋼）**を、**断面ビューポートでは記号**を見せる運用です（後述）。

姉妹プロジェクト「**配筋**」（[vectorworks-plugin-rebar](https://github.com/min-nano/vectorworks-plugin-rebar)）が、この「鉄筋」PIO を 3D パスで多数配置して配筋全体を組み立てます。1 本ごとを独立した鉄筋オブジェクト（＝実体のある 3D ソリッド）にすることで、断面ビューポートが各鉄筋を個別に切断でき、複雑な形状でも断面表現が破綻しません。

## 表示記号（配筋標準図 KSE 2008）

呼び径ごとの断面の表示記号は下表によります。記号ソリッドの断面形状（塗り円・中空リング・帯）としてパスに押し出され、切断面に現れます。大きさは `呼び径 × MarkScale` を外径とした模式表現です。

| 呼び径 | 記号 | 呼び径 | 記号 |
|---|---|---|---|
| D10 | ●（塗り丸・小） | D29 | ⊗（リング＋×） |
| D13 | ×（斜め十字） | D32 | ◎（二重リング） |
| D16 | ⊘（リング＋斜線） | D35 | ⊕（リング＋十字） |
| D19 | ●（塗り丸・大） | D38 | ●⊕（塗り丸＋十字） |
| D22 | ○（リング） | D41 | ⊗（リング＋×） |
| D25 | ⊙（リング＋中心点） | | |

## インストール

後述の `main.py` を PIO のスクリプトとして登録すれば、初回リセット時に依存ライブラリも含めて自動的にインストールされるため、手動インストールは任意です。オフライン環境などで手動インストールする場合は、このリポジトリをクローンまたはダウンロードし、リポジトリのルートで以下を実行します。

**macOS**
```bash
pip install --target "$HOME/Library/Application Support/Vectorworks/2025/Python Externals" .
```

**Windows（コマンドプロンプト）**
```bat
pip install --target "%APPDATA%\Nemetschek\Vectorworks\2025\Python Externals" .
```

> VectorWorks のバージョンが異なる場合は `2025` の部分を実際のバージョン番号に置き換えてください。

## VectorWorks へのプラグイン登録

ツール > プラグイン > **プラグインマネージャ** で新規プラグインを作成します。

1. **種別**: 「3D パス図形」（3D Path Object）を選択し、名前を `鉄筋` にする。
2. **パラメータ**: 以下のパラメータを **名前を完全一致** させて追加する（名前は半角英数、別名は OIP に表示したい日本語ラベル）。

   | 名前 | 種別 | 既定値 | 別名の例 |
   |---|---|---|---|
   | `Bar` | 文字列 | `D13` | 呼び径 |
   | `MarkScale` | 数値 | `4` | 断面記号倍率 |
   | `SymbolClass` | 文字列 |（空） | 断面記号クラス |

3. **スクリプト**: `main.py` の内容をそのまま Python スクリプトとして貼り付ける。
4. ワークスペースの編集でツールをパレットに追加する。

登録したスクリプトはリセットのたびに GitHub の `main` ブランチの最新コミットを確認し、インストール済みのコミットと異なれば Python Externals フォルダへ自動的に更新インストールしてから本体を実行します（更新後は VectorWorks を再起動しなくても次のリセットから新しいコードが使われます）。インターネットに接続できない場合はインストール済みのバージョンをそのまま実行します。

## 使い方

1. ツールを選び、鉄筋の芯線を 3D パスで描く（直線でも折れ線でも可）。
2. `Bar`（呼び径、`D13` 形式）を OIP で設定する。
3. `SymbolClass` に、断面記号ソリッド用のクラス名を指定する（例: `鉄筋-断面記号`）。本体の丸鋼は PIO 本体に割り当てたクラスに従います。

### 断面ビューポートでの表示（クラス表示の切り替え）

本体（丸鋼）と断面記号は**別クラスの 3D ソリッド**として同時に生成されます。断面ビューポートは 3D ソリッドを実体として切断するので、**どの位置・向きで切っても記号が正しい位置に出ます**（斜め配筋・折れ・フックにも対応）。

見せ分けは**ビューポートごとのクラス表示**で行います。

- **3D ビュー／モデル**: `SymbolClass`（断面記号クラス）を**非表示**にする → 丸鋼だけが見える。
- **断面ビューポート**: PIO 本体のクラス（丸鋼）を**非表示**にし、`SymbolClass` を**表示**にする → 切断面に記号だけが出る。

> 2D コンポーネントは使いません（オブジェクトのローカル 6 軸・紙面平面という制約と表示位置ずれの問題があるため、実体のソリッドをネイティブに切断する方式に変更しました）。

### 作図クラス

本体（丸鋼）と平面線は **PIO 本体の描画クラス**に、断面記号ソリッドは **`SymbolClass` で指定するクラス**に割り当てられ、線種・色・太さ・塗りはそれぞれのクラス属性に従います。クラスを使い分けることで、ビューごとの表示切り替えと見た目の管理ができます。

## 開発

テストは VectorWorks の公式 `vs.py` スタブをモック対象として `pytest` で実行します（`.github/workflows/test.yml` 参照）。

```bash
pip install -e ".[test]"
curl -fsSL https://raw.githubusercontent.com/Vectorworks/developer-scripting/main/Python/pages/files/vs.py -o tests/vs.py
mypy
pytest tests/ -v
```
